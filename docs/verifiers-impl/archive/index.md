# Archive

The archive preserves superseded notes, old working material, and handoff context for provenance.

Use it only after checking the current verifier sources. Archived material can explain how the work evolved, but it should not be treated as current implementation guidance unless it is explicitly confirmed by [`current-state.md`](../current-state.md), [`roadmap.md`](../roadmap.md), or a newer decision record.

## Contents

- [`implementation-log/`](implementation-log/index.md) — weekly engineering history.
- [`meeting-notes/`](meeting-notes/index.md) — advisor/thesis meeting history.
- [`investigations/`](investigations/index.md) — long technical investigations kept for provenance.
- [`runs/`](runs/index.md) — run-specific audits and one-off measurement notes.
- [`legacy-notes/README.md`](legacy-notes/README.md) — older verifier notes.
