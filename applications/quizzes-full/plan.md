### 8. Tournament

**Write functionalities:**
- `CreateTournament(executionId, creatorId, topicIds, numberOfQuestions, startTime, endTime)` ŌĆö create tournament + associated Quiz; saga fetches Execution (P4a: CREATOR_COURSE_EXECUTION), User/creator, Topics (P3: TOPIC_COURSE_EXECUTION check); P4b: QUIZ_COURSE_EXECUTION_CONSISTENCY, START_TIME_AVAILABLE_DATE/END_TIME_CONCLUSION_DATE, NUMBER_OF_QUESTIONS/QUIZ_TOPICS
- `AddParticipant(tournamentId, executionId, userId)` ŌĆö enroll participant; P4a: PARTICIPANT_COURSE_EXECUTION (fetch student by executionId+userId)
- `SolveQuiz(tournamentId, userId)` ŌĆö links a participant's existing QuizAnswer to the Tournament; saga reads Tournament (to obtain `quizAggregateId`), reads QuizAnswer by `(quizId, userId)`, then writes `quizAnswerAggregateId` + `quizAnswerVersion` onto the Tournament participant; enables the `QuizAnswerQuestionAnswerEvent` subscription for that participant
- `UpdateTournament(tournamentId, startTime, endTime, topicIds)` ŌĆö update timing or topics before `startTime`; saga fetches Quiz for QUIZ_COURSE_EXECUTION_CONSISTENCY (P4b); updates Quiz dates via command
- `CancelTournament(tournamentId)` ŌĆö cancel open tournament
- `DeleteTournament(tournamentId)` ŌĆö delete cancelled or finished tournament

**Read functionalities:**
- `GetOpenTournaments(executionId)` ŌĆö list open tournaments for an execution
- `GetTournamentById(tournamentId)` ŌĆö fetch tournament

**Events published:** ŌĆö

**Cross-aggregate prerequisites (P4a):**
- `CREATOR_COURSE_EXECUTION` ŌåÆ `CreateTournamentFunctionalitySagas` data-assembly step (fetch student by `(executionId, creatorId)` via `GetStudentByExecutionIdAndUserIdCommand`; throws if not enrolled)
- `PARTICIPANT_COURSE_EXECUTION` ŌåÆ `AddParticipantFunctionalitySagas` data-assembly step (same pattern)

- `START_TIME_AVAILABLE_DATE / END_TIME_CONCLUSION_DATE` ŌĆö same `startTime`/`endTime` passed as `availableDate`/`conclusionDate` to both in `CreateTournamentFunctionalitySagas`
