## Build System

### Key Commands

```bash
npm install                  # Install dependencies
npm run langium:generate     # Regenerate parser from grammar
npm run build                # Compile TypeScript + bundle + copy templates
npm run watch                # Watch mode (auto-rebuild)
```

### Build Pipeline

1. `npm run langium:generate` reads `nebula.langium` and generates `src/language/generated/` (AST types, parser, module)
2. `npm run build` runs:
   - `tsc -b tsconfig.src.json` — compiles TypeScript to `out/`
   - `node esbuild.mjs` — bundles for CLI distribution
   - `cp -r src/cli/templates out/cli/` — copies Handlebars templates

### Important: Grammar Changes

After modifying `nebula.langium`, always run both:
```bash
npm run langium:generate && npm run build
```

`langium:generate` regenerates the AST types. Without it, TypeScript won't see new grammar rules.

## Component Overview

### Language Layer (`src/language/`)

- **`nebula.langium`** — defines the DSL syntax: what users can write in `.nebula` files
- **`generated/ast.ts`** — TypeScript interfaces for every grammar rule (auto-generated, never edit)
- **`validation/`** — semantic checks that run during parsing (entity uniqueness, invariant syntax, method signatures)

### Engine Layer (`src/cli/engine/`)

- **`code-generator.ts`** — the main orchestrator. Parses `.nebula` files, validates, iterates aggregates, calls feature facades, writes output files
- **`generator-registry.ts`** — factory for all generator instances. Lazily creates and caches generators
- **`project-setup.ts`** — creates output directory structure, derives project name from path

### Feature Layer (`src/cli/features/`)

Each feature facade coordinates one "slice" of generation for an aggregate:

| Feature | What it produces |
|---------|-----------------|
| `EntityFeature` | JPA entities, DTOs, factories, repositories |
| `ServiceFeature` | Service class with CRUD + custom methods |
| `EventsFeature` | Event classes, handlers, subscriptions |
| `CoordinationFeature` | Functionalities (orchestration), event processing |
| `WebApiFeature` | REST controllers, request DTOs |
| `SagaFeature` | Saga entities, DTOs, states, CRUD saga workflows |

### Generator Layer (`src/cli/generators/`)

Generators contain the actual code-generation logic. They read from the AST and produce Java source strings. Organized by domain:

- **`microservices/`** — entity, service, events, repository, command generators
- **`coordination/`** — functionalities, web API controllers, configuration, top-level workflows
- **`sagas/`** — saga aggregates, saga CRUD operations, saga states

### Template Layer (`src/cli/templates/`)

Handlebars templates for structured Java output. Generators build a context object and render templates via `TemplateManager`. Some generators use inline string building instead of templates for simpler outputs.

### Utils (`src/cli/utils/`)

Shared utilities used across generators:

| File | Purpose |
|------|---------|
| `aggregate-helpers.ts` | Extracts entities, methods, events, repository from AST. Synthesizes `webApiEndpoints` from `@PostMapping` annotations. Cross-file type inference via model registry. |
| `file-writer.ts` | Writes generated files to disk with directory creation |
| `template-manager.ts` | Loads and caches Handlebars templates |
| `string-utils.ts` | Capitalize, lowercase, pluralize |
| `config-loader.ts` | Reads `nebula.config.json` for base package and framework version |
| `path-builder.ts` | Builds Java package paths per aggregate |
