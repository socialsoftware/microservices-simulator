# Engineering implementation log

The weekly implementation log is the engineering timeline for verifier work.

Use it to answer what changed in a specific week, what was validated, what was deferred, and what follow-up remained at that point. It is historical evidence, not the current source of truth. For present-tense status, return to [`current-state.md`](../../current-state.md).

## How to use this section

- Read the newest week first when you need recent changes.
- Use older weeks to reconstruct provenance and implementation sequence.
- Treat each week as a point-in-time record; later logs and [`current-state.md`](../../current-state.md) may supersede it.
- Use [`meeting notes`](../meeting-notes/index.md) for advisor-facing framing rather than engineering detail.
