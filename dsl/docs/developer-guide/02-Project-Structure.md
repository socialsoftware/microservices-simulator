## Core Directories Deep Dive

### 1. `src/language/` - Grammar and Parser

**Purpose:** Defines the Nebula DSL grammar using Langium framework.

```
src/language/
├── nebula.langium              # Grammar definition (500 lines)
├── nebula-module.ts            # Langium service setup
├── nebula-validator.ts         # Semantic validation
└── generated/                  # Auto-generated by Langium
    ├── ast.ts                  # AST type definitions
    ├── grammar.ts              # Parser implementation
    └── module.ts               # Service module
```

**Key Files:**

- **`nebula.langium`** - The heart of the DSL. Defines all language constructs (Aggregate, Entity, Property, etc.). Modified when adding new syntax.

- **`generated/ast.ts`** - TypeScript interfaces for every grammar rule. Auto-generated by `npm run langium:generate`. Never edit manually.

**Example AST Interface (auto-generated):**
```typescript
export interface Aggregate extends AstNode {
    name: string;
    annotations?: Annotation[];
    generateCrud?: boolean;
    aggregateElements: AggregateElement[];
    entities: Entity[];
}

export interface Entity extends AstNode {
    name: string;
    isRoot?: boolean;
    aggregateRef?: string;
    properties: Property[];
    invariants: Invariant[];
}
```

### 5. `src/cli/features/` - Feature Facades

**Purpose:** High-level orchestration of related generators. Each feature encapsulates a complete generation workflow.

**`entity-feature.ts` example:**
```typescript
export class EntityFeature {
    static async generateCoreComponents(
        aggregate: Aggregate,
        aggregatePath: string,
        options: GenerationOptions,
        generators: GeneratorRegistry
    ): Promise<void> {
        // 1. Generate entities
        for (const entity of aggregate.entities) {
            const entityCode = await generators.entityGenerator.generateEntity(entity, options);
            await FileWriter.writeGeneratedFile(...);
        }

        // 2. Generate DTOs
        const dtoCode = await generators.dtoGenerator.generateDto(entity, options);
        await FileWriter.writeGeneratedFile(...);

        // 3. Generate factory
        const factoryCode = await generators.factoryGenerator.generateFactory(aggregate, options);
        await FileWriter.writeGeneratedFile(...);

        // 4. Generate repository
        const repositoryCode = await generators.repositoryGenerator.generateRepository(aggregate, options);
        await FileWriter.writeGeneratedFile(...);
    }
}
```

**Benefits:**
- Single entry point per feature
- Coordinates multiple generators
- Handles file writing and error reporting
- Reduces complexity in main code generator

## Build System

### TypeScript Configuration (`tsconfig.json`)

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "outDir": "out",
    "rootDir": "src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "out", "src/language/generated"]
}
```

### Langium Configuration (`langium-config.json`)

```json
{
  "projectName": "Nebula",
  "languages": [
    {
      "id": "nebula",
      "grammar": "src/language/nebula.langium",
      "fileExtensions": [".nebula"],
      "textMate": {
        "out": "syntaxes/nebula.tmLanguage.json"
      }
    }
  ],
  "out": "src/language/generated"
}
```

**What Langium generates:**
- `src/language/generated/ast.ts` - AST type definitions
- `src/language/generated/grammar.ts` - Parser implementation
- `src/language/generated/module.ts` - Service module
- `syntaxes/nebula.tmLanguage.json` - VSCode syntax highlighting

### Build Scripts (`package.json`)

```json
{
  "scripts": {
    "build": "tsc -b tsconfig.src.json && node esbuild.mjs && cp -r src/cli/templates out/cli/",
    "watch": "concurrently -n tsc,esbuild -c blue,yellow \"tsc -b tsconfig.src.json --watch\" \"node esbuild.mjs --watch\"",
    "langium:generate": "langium generate",
    "langium:watch": "langium generate --watch"
  }
}
```

**Build pipeline:**
1. `langium:generate` - Generate parser from grammar
2. `tsc` - Compile TypeScript to JavaScript (`src/` → `out/`)
3. `esbuild` - Bundle VSCode extension
4. `cp` - Copy templates to output directory

**Why copy templates?**
Templates are not TypeScript files, so they're not processed by `tsc`. The build script explicitly copies them to `out/cli/templates/` so generators can find them at runtime.

### `bin/cli.js` - CLI Entry Point

```javascript
#!/usr/bin/env node

import { Command } from 'commander';
import { CodeGenerator } from '../out/cli/engine/code-generator.js';

const program = new Command();

program
    .name('nebula-cli')
    .description('Nebula DSL code generator')
    .version('0.0.1');

program
    .command('generate')
    .argument('<source>', 'Path to .nebula files')
    .option('-o, --destination <path>', 'Output directory')
    .option('-n, --name <name>', 'Project name')
    .action(async (source, options) => {
        await CodeGenerator.generateCode(source, options);
    });

program.parse();
```

cd dsl/nebula

**Features:**

For development, use watch mode to automatically rebuild on changes:

## File Naming Conventions

### DSL Files
- Lowercase aggregate names: `user.nebula`, `execution.nebula`
- Special files: `shared-enums.nebula`, `exceptions.nebula`

### Generated Java Files
- Entities: `User.java`, `ExecutionCourse.java`
- DTOs: `UserDto.java`, `CreateUserRequestDto.java`
- Services: `UserService.java`
- Controllers: `UserController.java`
- Factories: `UserFactory.java`
- Repositories: `UserRepository.java`, `UserCustomRepository.java`
- Events: `UserDeletedEvent.java`

### TypeScript Generator Files
- Generators: `*-generator.ts` (e.g., `entity-generator.ts`)
- Features: `*-feature.ts` (e.g., `entity-feature.ts`)
- Utilities: `*-helpers.ts`, `*-utils.ts`
- Builders: `*-builder.ts` (e.g., `event-context-builder.ts`)

## Common File Locations

### Finding Files by Purpose

| Purpose | Location |
|---------|----------|
| Add new grammar rule | `src/language/nebula.langium` |
| Create new generator | `src/cli/generators/<category>/` |
| Add template | `src/cli/templates/<category>/` |
| Modify generation pipeline | `src/cli/engine/code-generator.ts` |
| Add feature facade | `src/cli/features/` |
| Add helper utility | `src/cli/utils/` |
| View generated code | `applications/<project>/src/main/java/` |
| Test DSL | `dsl/abstractions/answers/` |

### Frequently Modified Files

When adding a new DSL feature, you'll typically modify:

1. `src/language/nebula.langium` - Add grammar rule
2. `src/cli/generators/<category>/*-generator.ts` - Implement generator
3. `src/cli/templates/<category>/*.hbs` - Create template
4. `src/cli/features/*-feature.ts` - Update feature facade
5. `src/cli/engine/generator-registry.ts` - Register generator (if new)
