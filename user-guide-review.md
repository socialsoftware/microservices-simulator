# User Guide Documentation Review

Full review of `dsl/docs/user-guide/` (chapters 01–11). Each example was generated and compiled from a fresh clone to verify all claims.

---

## Chapter 02: Getting Started

### Issues Found

1. **CRITICAL: No mention of which branch to use.** The `dsl/` directory only exists on the `nebula-new-simulator-version` branch. A fresh clone lands on `master`, which has no `dsl/` directory. The very first step fails silently. The docs need `git checkout nebula-new-simulator-version` before anything else.

2. **Maven version is oddly specific.** Docs say "Maven 3.9.9 or higher" — this is an unusual minimum patch version. It works fine with 3.9.11. Consider just saying "Maven 3.9+" or "Maven 3.9.x".

3. **VSCode extension path is wrong.** The docs say extensions are in `dsl/nebula/` (line 268 references `dsl/nebula/nebula-0.0.8.vsix`). They're actually in `dsl/extensions/` (`dsl/extensions/nebula-extension-0.0.2.vsix`). The version number is also wrong (0.0.2, not 0.0.8).

4. **`npm run release:minor` and `npm run release:major` don't exist.** The `package.json` only has `"release"` (which runs `bash release.sh patch`). The docs claim `release:minor` and `release:major` scripts exist.

5. **Expected output for tutorial generation is missing auto-generated query messages.** The actual output includes `✨ Auto-generated query for findByGenre` and `findByAvailableTrue` lines not shown in the docs.

6. **`-o` flag behavior is not explained.** The CLI auto-creates a subdirectory named after the project (e.g., `-o ../docs/examples/generated` creates `generated/01-helloworld/`). This is never explained and could confuse users.

## Chapter 03: Your First Aggregate

### Issues Found

7. **REST endpoint paths are wrong.** Docs claim standard REST:
   - `POST /tasks` → actual is `POST /tasks/create`
   - `PUT /tasks/{id}` → actual is `PUT /tasks` (ID in body)
   - `GET/DELETE /tasks/{id}` → actual uses `{taskAggregateId}`, not `{id}`

8. **`createTask` code snippet is materially incorrect.** Docs show `taskFactory.createTask(null, dto)`. Actual code generates an aggregate ID via `aggregateIdGeneratorService.getNewAggregateId()`, creates a `TaskDto` from the request DTO, and passes `(aggregateId, taskDto)` to the factory.

9. **Missing `nebula.config.json` requirement.** The chapter never mentions this required file.

10. **Directory tree shows `shared/enums/` but it's not generated** for this example (no enums defined).

11. **Saga generation description is incomplete.** Docs say sagas are generated for "create, update, and delete." Actual output includes saga workflows for read operations too (`GetTaskByIdFunctionalitySagas`, `GetAllTasksFunctionalitySagas`).

12. **Validation files are undercounted.** Tree shows `(+ annotations)` vaguely; there are actually 4 files in the validation directory.

## Chapter 04: Types, Enums & Properties

### Issues Found

13. **HIGH: Default values are NOT applied in generated entities.** The docs claim `Boolean available = true` generates `private Boolean available = true;`. The actual generated field is `private Boolean favorite;` with **no default value initializer**. Defaults are silently dropped.

14. **Generated directory structure is wrong in all chapters.** The docs describe `coordination/` and `sagas/` as top-level directories next to `microservices/`. In reality, everything is nested under `microservices/{aggregate}/coordination/` and `microservices/{aggregate}/aggregate/sagas/`. This is a pervasive error across chapters 3–10.

15. **Command handler classes are generated but never documented anywhere.** A `commandHandler/` directory with command classes is generated for every aggregate but never mentioned in any chapter.

16. **Events are auto-generated by `@GenerateCrud` without explanation.** `DeletedEvent` and `UpdatedEvent` classes are generated even without an `Events` block. The docs never explain this behavior.

## Chapter 05: Business Rules & Repositories

### Issues Found

17. **`@Query` and `@Param` shown on wrong file.** The docs show `@Query` and `@Param` annotations on `ProductCustomRepository.java`. They're actually on `ProductRepository.java`. `ProductCustomRepository` is a plain interface with no annotations.

18. **`@Param` is not generated at all.** The docs show `@Param("maxPrice")` but the generated code has just `Double maxPrice` without any `@Param` annotation.

19. **Spring Data convention methods are not convention-based.** Docs say `findByActiveTrue()` and `findBySku()` use "Spring Data naming conventions." The actual generated code has explicit `@Query` JPQL annotations on all methods, not relying on name parsing.

20. **Docs tell users to check the wrong file.** The walkthrough says "check `ProductCustomRepository.java` for the query methods" but the queries are in `ProductRepository.java`.

21. **`ProductRepository.java` is never mentioned.** The main Spring Data JPA repository file is omitted from the documentation entirely.

## Chapter 06: Cross-Aggregate References

### Issues Found

22. **"Two auto-generated fields" is actually three.** Docs say cross-aggregate entities get `aggregateId` and `version`. They also get `state` (`AggregateState`). The docs' own code example shows `teacherState` but the prose says "two fields."

23. **Empty mappings description omits `state`.** Same issue — says "just `teacherAggregateId` and `teacherVersion`" but `teacherState` is also generated.

24. **Collection service method names are wrong.** Docs claim `addTeacherToEnrollment` and `removeTeacherFromEnrollment`. Actual: `addEnrollmentTeacher` and `removeEnrollmentTeacher`. Also, 5 methods are generated (add, addBulk, get, remove, update) but docs only mention 2.

25. **"JPA embeddable" is wrong.** Step 4 says cross-aggregate entities generate a "JPA embeddable." They're actually full `@Entity` classes with `@Id` and `@GeneratedValue`. The docs' own code block shows `@Entity`, contradicting the prose.

26. **Missing event publisher.** Enrollment subscribes to `CourseDeletedEvent from Course`, but Course has no `Events` block publishing `CourseDeletedEvent`. This is a broken example.

## Chapter 07: Events & Reactive Patterns

### Issues Found

27. **HIGH: Event publishing API is wrong.** Docs show `unitOfWork.registerEvent(event)`. Actual: `unitOfWorkService.registerEvent(event, unitOfWork)` — events are registered via the **service**, not the UnitOfWork directly.

28. **Delete event code is wrong.** Docs show `event.setName(author.getName())` being called. Actual delete code only passes `aggregateId` to the event constructor — field setters are not called.

29. **`setPublisherAggregateVersion()` is never mentioned.** This is always called in generated code but docs don't show it.

30. **Event flow mechanism is wrong.** Docs describe direct method calls (`PostEventHandling.handleAuthorUpdated(event)`). Actual mechanism is `@Scheduled(fixedDelay = 1000)` polling with `eventApplicationService.handleSubscribedEvent(...)`. No direct event dispatch.

31. **PostEventProcessing and PostEventHandling roles are swapped.** Docs say PostEventProcessing does "routing" and PostEventHandling does "handlers." It's the opposite.

32. **HIGH: Inter-invariant cascade delete is NOT implemented.** `processAuthorDeletedEvent` is generated as an empty stub with a comment `// Reference constraint event processing - implement constraint logic`. The docs claim it works automatically.

33. **Many generated event files are unmentioned.** Subscription classes, handler classes, auto-generated publish events, and event handler base classes are all generated but never documented.

## Chapter 08: Tutorial — Library System

### Issues Found

34. **DSL line count is wrong.** Docs claim "~60 lines total." Actual: 102 lines across 4 files. Nearly double.

35. **Internal contradiction: intro vs table.** Intro says "50+ Java files, ~4,500 lines." Table says "120+ files, ~5,100 lines." The table is closer to reality (124 files).

36. **Code reduction ratio is wrong.** Claims "~85x" but 5,134 / 102 ≈ 50x (because DSL lines are 102, not 60).

37. **Generated structure diagram is wrong.** Shows `coordination/` and `sagas/` as top-level directories (same pervasive error). All per-aggregate file counts are wrong too.

38. **Missing `nebula.config.json` in walkthrough.** The step-by-step tells users to create 4 `.nebula` files but never mentions the required config file.

39. **Many generated file categories are completely omitted.** `command/`, `commandHandler/`, `coordination/validation/`, `aggregate/sagas/`, resource files — none mentioned.

## Chapter 09: Advanced Patterns

### Issues Found

40. **Answers project aggregate count is wrong.** Says "9 aggregates" but there are 8 (shared-enums and exceptions are not aggregates).

41. **TeaStore project aggregate count is wrong.** Says "6 aggregates" but there are 5 (shared-enums is not an aggregate).

42. **TeaStore Product "availability tracking" claim is false.** The actual teastore `product.nebula` has no `available` field. It has `name`, `description`, and `listPriceInCents`.

43. **TeaStore Order "order status tracking" claim is misleading.** The Order root entity has no `status` field.

44. **"Dto Entity" described as "embedded" but generates a separate JPA entity.** Docs say Dto Entities "are embedded within the root entity." They actually generate full `@Entity` classes with their own tables.

45. **Advanced example doesn't demonstrate custom services/endpoints.** The chapter documents `Service` and `WebAPIEndpoints` custom blocks but the 07-advanced example doesn't use either feature.

## Chapter 10: Generated Code

### Issues Found

46. **Entity class is `abstract` but not shown as such.** Docs show `public class User extends Aggregate`. Actual: `public abstract class Member extends Aggregate`.

47. **Service code examples are significantly wrong.** Create method uses `null` for aggregateId (wrong), update method signature has separate `id` parameter (wrong), no error handling shown (always generated), no event publishing shown (always generated).

48. **Controller code examples are wrong.** Class-level `@RequestMapping` shown but not generated. Endpoint paths differ (`@PostMapping` vs `@PostMapping("/members/create")`). Path variable naming wrong. Missing `@ResponseStatus` annotations.

49. **Event publishing pattern is wrong (same as ch07).** Shows `unitOfWork.registerEvent(event)` instead of `unitOfWorkService.registerEvent(event, unitOfWork)`.

50. **Resources directory listing is incomplete.** Shows only `application.yml`. Actual: `application.properties`, `application.yml`, `database.properties`, `logback-spring.xml`.

## Chapter 11: Reference

### Issues Found

51. **CustomEndpoint grammar omits `throwsException` field.** The actual grammar includes `('throwsException' ':' throwsException=('true' | 'false'))?` which is undocumented.

52. **Method expressions table is incomplete.** Missing: `count`, `distinct`, `findAny`, `findFirst`, `getState`, `allMatch`, `anyMatch`, `noneMatch`, `isEqual`, `now()`.

53. **Method, Workflow, and Functionalities listed but not documented.** They appear in the AggregateElement table but have no grammar snippets or syntax reference.

54. **Reserved keywords list is missing ~30+ keywords.** Missing: `method`, `workflow`, `causal`, `saga`, `step`, `compensation`, `call`, `extract`, `where`, `registerState`, `final`, `dto-exclude`, `message`, `void`, etc.

55. **Complete example uses "answers" domain, not tutorial domain.** Breaks consistency with the rest of the user guide.

## Overall Assessment

The docs cover the right concepts and the DSL examples are all syntactically correct — all 7 generate and compile successfully. However, the **generated code descriptions are consistently wrong** — they appear to describe a simplified/idealized version of what the generator produces, not the actual output. The directory structure description is wrong in every chapter. The tutorial walkthrough is missing the `nebula.config.json` step. And the docs assume the user is on the right branch without ever saying so.
