## Build System

### Build Scripts (`package.json`)

```json
{
  "scripts": {
    "build": "tsc -b tsconfig.src.json && node esbuild.mjs && cp -r src/cli/templates out/cli/",
    "watch": "concurrently -n tsc,esbuild -c blue,yellow \"tsc -b tsconfig.src.json --watch\" \"node esbuild.mjs --watch\"",
    "langium:generate": "langium generate",
    "langium:watch": "langium generate --watch"
  }
}
```

**Build pipeline:**
1. `langium:generate` -- Generate parser from grammar
2. `tsc` -- Compile TypeScript to JavaScript (`src/` вҶ’ `out/`)
3. `esbuild` -- Bundle VSCode extension
4. `cp` -- Copy templates to output directory (templates are not TypeScript, so `tsc` doesn't process them)

## Generation Pipeline

The pipeline has 6 phases:

### Phase 1: Discovery

Scan the input directory for `.nebula` files.

**File:** `src/cli/utils/file-utils.ts`

### Phase 2: Parsing

Load files into Langium workspace, build AST, resolve cross-references, register all models in the global registry.

**File:** `src/cli/engine/code-generator.ts`

```typescript
// After parsing, register models for cross-file type inference
registerAllModels(models);
```

### Phase 3: Validation

Semantic validation: structural checks (root entity exists, one Repository per aggregate), type checks, and cross-reference validation.

### Phase 4: Configuration

Setup output directories, load optional `nebula.config.json`, initialize generator registry, build DTO schema registry.

**File:** `src/cli/engine/project-setup.ts`

### Phase 5: Generation

The main loop iterates over all aggregates and delegates to feature facades:

```typescript
for (const model of models) {
    for (const aggregate of model.aggregates) {
        await EntityFeature.generateCoreComponents(aggregate, ...);
        await ServiceFeature.generateService(aggregate, ...);
        await EventsFeature.generateEvents(aggregate, ...);
        await CoordinationFeature.generateCoordination(aggregate, ...);
        await WebApiFeature.generateWebApi(aggregate, ...);
        await ValidationFeature.generateValidation(aggregate, ...);
        await SagaFeature.generateSaga(aggregate, ...);
    }
}
```

### Phase 6: Completion

Write project files (`pom.xml`, `.gitignore`, `application.yml`), generate configuration files, print statistics.

## Code Generator Orchestration

**File:** `src/cli/engine/code-generator.ts`

This is the main entry point. It:
1. Creates Langium services
2. Discovers and parses `.nebula` files
3. Validates models
4. Sets up output directories
5. Iterates over aggregates, calling feature facades
6. Writes project-level files

## Model Registry

After parsing, all models are registered globally for cross-file type inference:

```typescript
let globalModelRegistry: Model[] = [];

export function registerAllModels(models: Model[]) {
    globalModelRegistry = models;
}

export function findAggregateInModels(aggregateName: string): Aggregate | undefined {
    for (const model of globalModelRegistry) {
        const aggregate = model.aggregates.find(a => a.name === aggregateName);
        if (aggregate) return aggregate;
    }
    return undefined;
}
```

This enables generators to resolve types from referenced aggregates in different files.

## Error Handling

**File:** `src/cli/utils/error-handler.ts`

Errors have context (aggregate, generator, operation) and severity levels:

- **FATAL** -- Stop generation immediately
- **ERROR** -- Report error but continue
- **WARNING** -- Log warning and continue
- **INFO** -- Informational message

```typescript
ErrorHandler.handle(
    error,
    ErrorUtils.aggregateContext('generate entity', aggregate.name, 'entity-generator'),
    ErrorSeverity.FATAL
);
```

## Design Philosophy

### Convention Over Configuration
Smart defaults with override capability: service names auto-generated, CRUD operations generated with `@GenerateCrud`, standard package structure.

### Type Safety
Strong typing throughout: grammar enforces valid constructs, type inference reduces boilerplate, generated code is type-safe Java.

### Separation of Concerns
Clear boundaries: grammar (syntax) вҶ’ validators (semantics) вҶ’ generators (code) вҶ’ templates (structure).

### Pipeline Architecture
Modular, composable generators registered in a central registry with dependency tracking.
