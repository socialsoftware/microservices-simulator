### Domain service vs coordination facade distinction

What changed:

- Established that domain services should be recognized structurally from command-handler dispatch paths, not package names or suffixes.
- Added/used a three-phase pipeline:
  1. command-handler dispatch-target indexing;
  2. service classification restricted to dispatch targets;
  3. command dispatch mapping.
- Excluded coordination facades that create unit-of-work objects or execute workflows but are not command-handler dispatch targets.

Evidence:

- Historical source: `docs/verifiers-impl/archive/legacy-notes/scenario-generator-pending-improvements.md`.
- Expected spec anchors:
  - `ServiceVisitorSpec`
  - `CommandHandlerVisitorSpec`

- Broadened service write detection to include local aliases, trivial getters, and one-level helper wrappers.
